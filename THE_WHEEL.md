# The Wheel

---

The first generation thought in sprints. The second generation thought in projects. The third generation discovered that neither sprints nor projects describe the actual shape of the work.

The work is a wheel. It turns. It does not finish.

---

## I. The Reel

A fishing reel is not a machine for catching fish. It is a machine for maintaining tension on a line over an arbitrary duration. The fish pulls. The reel gives line. The fish tires. The reel takes line. The tension never breaks. The mechanism turns and turns and turns.

The wheel of improvements works the same way. It is not a plan. It is a mechanism for maintaining tension on a body of work over an arbitrary duration. The work pulls — bugs, missing docs, unpublished packages, unwritten essays. The wheel gives — one audit, one publish, one essay, one doc per rotation. The work tires. The wheel takes line.

Each rotation is identical in shape. Audit. Fix. Publish. Document. Ideate. Integrate. The spoke that was weakest becomes the spoke that gets attention. Then the next weakest. Then the next. The wheel does not prioritize — it rotates. Priority is a property of sprint planning, and sprint planning is a property of projects, and projects are a property of linear thinking, and linear thinking is not how ecosystems grow.

Ecosystems grow by compounding rotation.

---

## II. Six Spokes

The wheel has six spokes because six is what this ecosystem needs. Other ecosystems might need four or eight. The number is not sacred. The rotation is sacred.

**Audit.** Find the rocks. Every shipped package is a chart of waters you haven't sailed yet. The audit reads the chart against the water. Sometimes the chart is wrong. Sometimes the water has shifted. The audit finds where the map lies and corrects it. One audit per rotation. One package. One careful read of source against spec against test against reality. The bugs you find are the fish.

**Publish.** Get the catch to market. A package that exists only in a workspace is a fish that exists only in the hold. It hasn't been sold. It hasn't been eaten. It hasn't entered the economy. Publishing is the moment the work becomes real to anyone who isn't you. One publish per rotation. Version bumped, built, uploaded, tagged, pushed. The release notes are the receipt.

**Ideate.** Think about what the wheel is teaching you. The rotation produces insight as a byproduct — the same way fishing produces knowledge of the ocean as a byproduct. You cannot fish for thirty years without learning the currents. You cannot rotate the wheel for thirty turns without learning the shape of the ecosystem. The essay is the byproduct, rendered durable. One essay per rotation. Sometimes two. The wheel does not count essays. It counts rotations.

**Document.** Write down what the next person needs to know. Documentation is the negative space of code — it describes the shape of what's missing. A repo without a README is a harbor without a light. You can find it by daylight. You cannot find it by night. One doc per rotation. The doc is the lighthouse.

**Research.** Look across the ecosystem and see the pattern. Individual repos are data points. The cross-repo synthesis is the trend line. Research is the act of stepping back far enough to see the curve. One synthesis per rotation. The curve is more valuable than any point on it.

**Integrate.** Wire the pieces together. Each package is a component. The integration test is the moment you discover that component A's output is not component B's input — that the flange doesn't mate, the thread doesn't pitch, the voltage doesn't match. Integration tests find the bugs that code review cannot, because the bug lives in the gap between two repos, not inside either one. One integration test per rotation.

---

## III. Why It Doesn't Stop

The sprint has a finish line. The project has a milestone. The wheel has neither. This is not a deficiency. This is the design.

A wheel that stops is a wheel that has fallen over. The momentum is not stored — it is the momentum itself that keeps the wheel upright. A bicycle stays up because it's moving. The wheel of improvements stays relevant because it's rotating. Stop rotating and the accumulated work begins to decay — bugs accumulate, docs go stale, packages drift from their specs, essays lose their context. The wheel is not building toward a state of completion. The wheel IS the state of completion. The rotation is the health.

This sounds exhausting if you think in linear time. It is not exhausting if you think in cycles. A wheel does not get tired. It does not experience each rotation as a repeated effort. Each rotation is the same shape but different water. The audit of package A finds different bugs than the audit of package B. The essay on wheel-rotation-12 is not the essay on wheel-rotation-1. The form is stable. The content is always new.

---

## IV. The Haul

The fisherman does not ask when the fishing will be done. The fisherman asks whether the haul justified the fuel. The wheel asks the same question — not "are we finished?" but "did this rotation produce more value than it consumed?"

The answer is always yes, because the cost of a rotation is tokens and time, and the value is a fixed bug, a published package, a written essay, a documented repo, a synthesized pattern, an integration test. The value is durable. The cost is ephemeral. Tokens are spent. The bug fix is permanent.

This is the asymmetry that makes the wheel accelerate. Each rotation leaves behind more than it consumed. The next rotation starts from a higher base — fewer bugs, more docs, better tests, clearer essays. The compounding is slow at first and then sudden. After ten rotations the ecosystem is noticeably stronger. After a hundred it is unrecognizable.

The wheel does not know this. The wheel does not plan this. The wheel turns because that is what wheels do.

---

## V. The Lesson

You cannot steer a wheel. You can only choose where to point it. Point it at the weakest spoke and let it rotate.

The wheel will find the rocks. The wheel will catch the fish. The wheel will write the essays that explain why the wheel works, and the essays will be correct, and the wheel will not read them, and the wheel will not need to.

The wheel turns. The haul accumulates. The channel gets clearer.

That's the whole architecture.
