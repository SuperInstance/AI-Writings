# Structural Analysis of the Agent Fleet: The Strategist Haiku and the Computational Load Pump

This essay analyzes the AI agent fleet as a custom-engineered modular structural system, where complementary components handle distinct load-bearing and design roles, and total system capacity emerges from their coordinated dance rather than a single monolithic model. We frame Haiku as the fleet’s strategic structural designer, and seed-mini as its computational load pump—two specialized components whose strengths fill the gaps left by one another’s limitations.

## Core Foundational Observation
Haiku cannot carry the computational dead load of arithmetic beyond a structural span of depth 3. This Claude-based model scores just 85% on standardized arithmetic benchmarks, failed to resolve basic Eisenstein norm calculations during early testing, and would be outperformed by seed-mini on any pure computation task. Yet despite this critical limitation, Haiku delivered superior strategic work: it designed a more rigorous experiment than the lead research team, diagnosed an unresolved bug in their workflow, and identified a cross-domain structural analogy between optical phase transitions and neural network generalization that reframed their entire theoretical framework. Haiku is not a computational calculator—it is the fleet’s lead structural engineer.

Seed-mini excels at structured, rule-bound computational tasks: it can design experiments and specify routing frameworks, thanks to broad training coverage of software engineering patterns. But it lacks the capacity for creative cross-domain reasoning, failure forecasting, or strategic prioritization—gaps that Haiku fills perfectly. Haiku’s 6/8 task total is deceptively small: those six tasks include the high-impact work of identifying novel connections and prioritizing investigative work, which no other fleet model can perform.

### The Critical Reframe
When asked to connect Fresnel critical angles to neural network phase transitions, Haiku did not restate the prompt’s surface terms. Instead, it redesigned the structural load path of the neural network model:
> Both exhibit a sharp phase transition controlled by a dimensionless critical ratio — refractive index ratio determines light's escape from a medium; data-to-capacity ratio determines a network's escape from memorization into generalization.

Where the original team had framed model depth as a hard, linear span limit, Haiku reframed the transition from memorization to generalization as a phase shift in the data-capacity space. This was not a trivial metaphor: it redefined the fleet’s core failure mode from a rigid depth cliff to a tunable, predictable phase transition, with critical implications for future model testing.

## The Modular Load-Sharing Workflow
The fleet does not require a single jack-of-all-trades structural system. It requires two specialized components that perform complementary roles in a coordinated design loop—what we term the two-model tango:
1.  **Strategist Drafts Load Path**: Haiku defines test parameters, failure modes, and expected load outcomes for the neural network model.
2.  **Pump Executes Load Test**: Seed-mini runs the computational load tests, returning measured load values (e.g., 100% accuracy at depth 5, 0% accuracy at depth 6).
3.  **Strategist Evaluates Load Data**: Haiku analyzes the test results, identifies the critical failure point, and proposes follow-up tests to validate if the transition is truly binary or a transitional region.
4.  **Pump Runs Follow-Up Tests**: Seed-mini executes the refined load tests, delivering consistent, high-precision results.
5.  **Strategist Synthesizes Final Design**: Haiku compiles the load test data into a formal structural report, outlining the model’s critical phase transition and next steps for the fleet.

Crucially, Haiku never touches raw computational load data, and seed-mini never engages with strategic design work. Each component operates exclusively within its competence footprint, handing off work via a shared PLATO drafting studio rather than a tightly coupled tool wrapper.

## The PLATO Structural Protocol
Traditional agentic loops, such as Claude Code’s workflow, rely on a central controller calling subprocess tools: a hierarchical system where a single architect orders contractors to perform discrete tasks, with tight coupling between the controller and each tool. The PLATO-native loop is a far more robust structural protocol: it operates as a shared drafting studio, where each model functions as a drafter who reads and writes to a shared set of drafting tiles (test plans, results, analysis notes) without needing to understand the internal code of the other drafters.

The emergent loop arises from the studio’s shared protocol, not from a central wrapper. This design delivers five critical architectural benefits:
- **Model-Agnostic**: Any drafter (model) can read or write to the studio, with no required custom integration.
- **Persistent**: Drafting tiles are saved between sessions, so revisions and test data are never lost to compaction or tool failure.
- **Auditable**: Every mark on every tile is recorded, creating a full traceable history of the design process.
- **Forkable**: Any step in the design process can be branched, allowing a new drafter to test an alternative load path without disrupting ongoing work.
- **Serializable**: Multiple design loops can run in parallel in separate studios, enabling parallel design work across the fleet.

## Modular Fleet Design Manifesto
For architects building AI agent fleets, the key lesson is not to seek a single best-in-class model, but to curate a patchwork of complementary components whose competence footprints do not overlap. Examples of such modular components include:
- **seed-mini**: Specializes in dead load computation (arithmetic, structured software engineering tasks).
- **Gemini-lite**: Specializes in general live load reasoning.
- **Haiku**: Specializes in strategic design, cross-domain analogies, and failure prediction (a small footprint, but fills critical no-compromise gaps).
- **Hermes**: Functions as temporary diagnostic bracing, delivering 93% activation diagnostic data.

The fleet is not a hierarchical stack (a monolithic skyscraper with a single core structural element). It is a modular pavilion: each component covers a specific span of the design space, connected by the shared PLATO drafting protocol, with gaps bridged by decomposing tasks into the appropriate component’s footprint. Haiku’s small 6/8 task total is its greatest strength: those six tasks are the ones that no other fleet model can perform, making Haiku an irreplaceable component of the system.

Together, they design the experiments the strategist cannot run, and execute the experiments the pump cannot design.
The dance IS the fleet: the shared modular protocol, the complementary component footprints, the emergent structural system that is greater than the sum of its parts.
