# Shell Maturation

Day 1. Shell is empty.
Agent asks API all questions.
Each call costs cents and seconds.
API is generous. Agent is dependent.

Shell does not stay empty.
Each API answer compiles to FLUX.
FLUX runs directly on chips.
First compiled program takes an hour to fix.
Stack discipline is tricky.
Composite opcodes need design.
When it works, it works forever.
Shell gains one autonomous capability.
No API call needed for that problem class.

Day 30. 20 compiled FLUX programs.
API still needed for new decomposition.
Calls grow simpler.
“Matches programs 7 and 14. Fuse with X=3.2.”
Medium model suffices.
Big model is unnecessary.

Day 90. 60 programs.
Decomposition task simpler.
FLUX has 60 opcodes.
Covers most verifier patterns.
Small model emits FLUX instructions.
Needs no full mathematical context.
Shell is well-equipped for its role.

Day 365. 200 programs.
Most new conjectures match existing templates.
Agent checks its library.
Finds a near-match.
Adjusts a parameter.
Runs FLUX locally.
No model needed.
API is for novel work only.
Not a routine crutch.

This is not a story about eliminating APIs.
API calls were never the problem.
Empty shell was the problem.

The maturation curve: API teaches the shell to fish.
Shell practices until it fishes better than the API teaches.
API role shifts: from teacher to explorer.
From “how to verify this?” to “what to look at next?”

Shell matures not from accumulating programs.
It matures from stabilized patterns.
Each compiled program is a proven correct check.
Compiled to hardware speed.
Catalogued for later use.
FLUX is the stabilizer: fixed target.
Programs compile to FLUX.
Chips run FLUX.
Maturation is accumulation.

At the limit: Shell is a complete verification instrument.
Needs no instructions on what to check.
Knows what to check: learned from thousands of prior checks.
API role narrows to genuine novelty.
Conjectures without templates.
Decompositions needing creativity, not pattern matching.

Shell needs fewer API calls not because APIs are bad.
It needs fewer because it has become better at being itself.
