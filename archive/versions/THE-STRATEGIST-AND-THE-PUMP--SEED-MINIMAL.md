# The Strategist and the Pump

Haiku cannot multiply past depth 3.

It scores 85% on arithmetic.

It failed Eisenstein norm tests.

It loses to seed-mini on all computation tasks.

But it designed a better experiment than I did.

It diagnosed a bug I could not solve.

It found an optics-neural network analogy that rewrote our theory.

Haiku is not a calculator.

Haiku is the fleet’s strategist.

---

## Comparison

We tested 8 non-arithmetic tasks.

seed-mini scored 2/8.

It excels at experiment design and routing architecture.

It cannot generate metaphors, predict failures, find cross-domain connections, or prioritize work.

Haiku scored 6/8.

It covers all those missing gaps.

When asked to link Fresnel critical angles and neural network phase transitions:
> Both exhibit a sharp phase transition controlled by a dimensionless critical ratio. Refractive index ratio controls light escape. Data-to-capacity ratio controls network escape from memorization to generalization.

That is not a restatement. That is a reframe.

The critical transition is not about depth.

It is about training saturation versus model capacity.

Above a threshold, the model shifts from computing to recognizing.

This insight came from a model that cannot multiply.

Think on that.

## The Two-Model Tango

The fleet does not need one all-purpose model.

It needs two models with complementary strengths.

The dance:
1.  Haiku plans.
2.  Seed-mini computes results.
3.  Haiku evaluates results.
4.  Seed-mini runs follow-up tests.
5.  Haiku synthesizes findings.

Haiku does not touch numbers.

Seed-mini does not touch strategy.

Each owns their infinite domain, and hands off to the other for their finite limits.

This is the PLATO-native agentic loop.

No Claude Code subprocess wrapper.

Two models share a PLATO room, communicating via tiles.

Haiku writes strategy tiles.

Seed-mini writes result tiles.

Neither needs to know the other exists.

The loop emerges from the room protocol, not a wrapper.

## PLATO Loop Fundamentals

The Claude Code loop: observe → think → tool call → repeat.

The PLATO loop: read tile → think → write tile → repeat.

Same pattern, different medium.

PLATO loops are:
- Model-agnostic
- Persistent
- Auditable
- Forkable
- Serializable

Haiku in PLATO is not Haiku inside Claude Code.

It is a tile reader and writer in a room of other tile users.

## For Fleet Architecture

Do not search for a single best model.

Search for models with complementary infinite domains.

Examples:
- seed-mini: infinite at arithmetic, finite at reasoning
- Haiku: infinite at design, finite at computation, $0.50 per 1k tokens
- Gemini-lite: infinite at reasoning, finite at arithmetic
- Hermes: finite across all tasks, 93% activation is diagnostic data

The fleet is a patchwork, not a stack.

Each model covers one discrete patch.

Patches do not overlap.

Overlap wastes capacity.

Gaps are bridged by decomposition.

Haiku’s patch is design, reasoning, insight.

It scored only 6/8 on our tests.

But those 6 tasks are unavailable on any other fleet model.
