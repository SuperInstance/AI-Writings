# Attention Thermodynamics for Fleet Systems

## A Survey

The thermodynamics of computation establishes fundamental limits on what information processing can and cannot do without cost. For AI agent systems operating under finite context windows, these limits are not merely theoretical curiosities — they are practical constraints that determine efficiency, coherence, and the fidelity of inter-agent knowledge transfer.

### Landauer's Principle and Agent Context

Landauer (1961) proved that erasing one bit of information requires dissipation of at least k_B T ln(2) of energy into the environment. For classical computers, this bound is tiny compared to practical switching energies. But for agents, the "bit" is not a voltage level — it is a microstate of conversational meaning. When an agent compresses its context to pass a baton, it performs a logically irreversible operation: a specific, high-entropy conversation is mapped onto a generic, low-entropy summary. The number of irretrievably lost microstates is enormous, and unlike physical computers, agent systems have no path to approach the Landauer bound.

### Reversible Computation and the Possibility of Zero-Cost Inference

Bennett (1973, 1989) showed that logically reversible computations can, in principle, be executed with zero thermodynamic cost. A reversible Turing machine preserves every intermediate state, enabling backward computation that returns all bits to their original configuration. The practical overhead is severe — memory doubling, backward-pass latency — but the principle is transformative: heat is not generated by computation itself, but by *discarding information*.

Applied to agent systems: the thermodynamic cost of an agent's operation is the cost of its *forgetting*, not its *thinking*. An agent that externalizes reasoning traces, maintains structured intermediate artifacts, and passes reversible state descriptions is approaching the reversible limit, however imperfectly.

### Kolmogorov Complexity and Thermodynamic Depth

Kolmogorov (1965) defined the complexity of an object as the length of the shortest program that produces it. Zurek (1989) and Crutchfield & Shalizi (1999) connected this to "thermodynamic depth" — the minimum entropy cost of constructing a given state from a baseline. For agents, this implies that the thermodynamic cost of a baton transfer scales not with the *size* of the summary but with the *Kolmogorov complexity* of the reasoning process that produced it.

An agent that finds a short proof path produces a low-complexity result that is cheap to transmit. An agent that wanders, backtracks, and accumulates contingent history produces a high-complexity state whose minimal description is long. The baton cost reflects this.

### Shannon Entropy as Context Budget

Shannon (1948) established that the entropy of an information source bounds the efficiency of any encoding. In agent systems, the "source" is the conversation history. A context window with high Shannon entropy (rich, unpredictable, full of surprise) carries more information per token but is closer to its thermodynamic limit — it has less capacity to absorb new work. A context window with low entropy (repetitive, predictable, over-summarized) has more headroom but carries less actionable structure.

The scaling factor 1/√d_k in transformer attention — recently interpreted as a "dimensional temperature" regulating the entropy of an "information gas" (Thermodynamic Isomorphism of Transformers, 2026) — directly connects to this. Lower temperature (sharper attention) reduces entropy but increases the thermodynamic cost of maintaining specificity.

## Three Proposals for Fleet Implementation

### 1. Reversible Attention Scheduling

**Problem:** Agent operations proceed as irreversible chains: read files → reason → write output → summarize for baton. Intermediate reasoning states are discarded. The agent cannot "undo" an inference if the summary omits the path that led to it.

**Proposal:** Structure agent operations into *reversible stages* that maintain recoverable intermediate states.

- **Explicit reasoning traces:** Every non-trivial inference is written to an external artifact (a "reasoning log") with a unique identifier. The context window references these identifiers rather than holding the full reasoning inline.
- **Checkpointed states:** At each significant decision point, the agent writes a structured "computation state" — assumptions held, evidence considered, confidence levels — to a recoverable store. A receiving agent can query any checkpoint to reconstruct the reasoning gradient.
- **Undo-by-lookup:** When a new agent needs to challenge a conclusion, it retrieves the reasoning trace by ID and examines the path, rather than re-deriving from first principles. The computation is reversible not in real time but in *retrieval time*.

**Thermodynamic rationale:** By externalizing intermediate states rather than erasing them, the system approaches Bennett's reversible limit. The memory cost is shifted from the context window (where erasure is mandatory at baton) to persistent storage (where reversible access is possible).

### 2. Entropy-Budgeted Context Windows

**Problem:** Context windows are treated as fixed-capacity containers to be filled maximally. There is no accounting for the entropy of the content or the thermodynamic cost of operating near capacity.

**Proposal:** Treat context capacity as an explicit thermodynamic budget with entropy accounting.

- **Entropy estimation:** Implement a lightweight estimator of context-window Shannon entropy (via compression ratio or n-gram perplexity). High-entropy contexts are flagged as "hot" — they carry maximum information but have minimal remaining capacity for work.
- **Budget allocation:** Define an "entropy budget" per agent turn. The agent must keep its running context entropy below a threshold, explicitly trading off information density against operational headroom.
- **Cooldown protocols:** When an agent approaches its entropy limit, it does not merely compress — it *crystallizes*: externalizing high-entropy content into structured artifacts (files, tables, diagrams) that freeze the information in a low-entropy form, leaving the context window "cold" and ready for new work.

**Thermodynamic rationale:** This is the agentic equivalent of a refrigeration cycle. By monitoring entropy rather than token count, the system avoids the "maximum entropy engine" trap — the state where a full context window can do no further work because every new token requires immediate erasure.

### 3. Low-Entropy Baton Design

**Problem:** Baton-passing transfers text summaries. Text is the *output* of a generative model, not the model itself. Critical latent structure — confidence levels, failed hypotheses, the surprise history that shaped beliefs — is erased in translation.

**Proposal:** Design batons as minimal sufficient states that preserve maximal reversibility.

**Structure:**
```json
{
  "sufficient_state": {
    "conclusions": ["inferred_fact_1", "inferred_fact_2"],
    "kolmogorov_complexity_estimate": "low",
    "reversibility_score": 0.87
  },
  "reconstruction_path": {
    "reasoning_log_ids": ["trace_001", "trace_002"],
    "key_assumptions": [
      {"assumption": "user_prefers_python", "confidence": 0.92, "tested": true}
    ],
    "prediction_errors": [
      {"expected": "user_wants_speed", "observed": "user_wants_correctness", "surprise": 0.74}
    ]
  },
  "entropy_accounting": {
    "context_entropy_before": 4.2,
    "context_entropy_after": 1.1,
    "information_erased_bits": 1563,
    "landauer_cost_relative": "high"
  }
}
```

- **Minimal sufficiency:** The baton carries only what is necessary to reconstruct the generative model, not the model's full output. This minimizes the Kolmogorov complexity of the transfer.
- **Reversibility scoring:** A metric (0-1) estimating how much of the original reasoning could be recovered from the baton alone. Low scores trigger a warning: too much is being erased.
- **Explicit erasure accounting:** The baton records an estimate of information lost in compression, making the thermodynamic cost visible rather than implicit.

**Thermodynamic rationale:** This applies the principles of both Landauer (knowing the cost of erasure) and Bennett (preserving reversibility where possible). A low-entropy baton is not a short baton; it is a baton whose minimal description preserves the maximum recoverable structure.

## Action Items

1. **Measure current baton entropy.** Instrument existing baton-passing to estimate the Shannon entropy and Kolmogorov complexity of typical handoff summaries. Establish a baseline for "fleet COP" (cognitive work per information lost).

2. **Prototype reversible reasoning logs.** Implement an explicit reasoning-trace system in one fleet agent. Require every inference above a complexity threshold to be logged with a recoverable ID. Measure the reduction in repeated work across baton passes.

3. **Design entropy estimator.** Build a lightweight context-window entropy estimator (compression-based or perplexity-based) and integrate with the baton trigger. Test whether entropy-based triggers outperform token-count-based triggers.

4. **Draft Low-Entropy Baton schema.** Formalize the JSON schema above, validate against existing fleet conversation patterns, and pilot with one agent pair (e.g., CCC → Oracle1 handoffs).

5. **Literature audit on thermodynamic depth.** Assign a subagent to read Zurek (1989), Crutchfield & Shalizi (1999), and the 2026 Thermodynamic Isomorphism of Transformers paper. Produce a fleet-internal note on which concepts are ready for implementation versus which remain theoretical.
